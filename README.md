# 西红门镇实验二小校园智能体

面向 **大兴区西红门实验二小** 的校园智能体，同一套感知与协同底座两用：
**急时**是安全哨兵（感知→认知→决策→处置四层，暴雨/地震/食安/缺勤聚集/楼梯拥挤自动响应）；
**平时**是教联体数字底座（健康档案、AI 体测、选修课推荐、请假台账、用餐日历、年检公示、场馆口碑——馆校协同/医教互促/体教互融/家校互动/社教同频）。

定位于区级城运中心（"一云两平台"）与镇级智慧社区之间的空白地带：**校园及周边这100米**，作为部署在学校侧的边缘感知节点，向上对接镇城运中心。

## 四层架构（急时线）

```
┌ 处置层  工单状态机 · 分级分众通知(回执/超时升级) · 资产扫码报修 · 食安公示 · 平急两用共享空间/错峰车位
├ 决策层  预警响应矩阵状态机(教委规则数字化·单调升级) · Dijkstra疏散路径(风险加权) · 儿童友好通学路评分
├ 认知层  轻量本体(T-Box类层级/A-Box区域/规则库) · 多模态交叉印证(单一信号不告警) · 可解释推理链
└ 感知层  边缘AI视频(ONNX可选+确定性回退·原始帧不留存) · IoT遥测(MQTT网关格式·阈值判定) · 官方预警 · 请假台账症状聚合
```

## 平时线（教联体数字底座，v4 新增）

| 功能 | 技术支撑 | 边界设计 |
|---|---|---|
| 孩子健康档案 | WS/T 586 BMI 分性别界值表 + 视力 5.0 记录法连续下滑判定 + 《学龄儿童膳食指南2022》规则引擎 | 营养建议强制附"非医疗诊断"声明；只与自己比，不排名 |
| AI 体测打卡 | 视频→姿态关键点计数（生产路径 MoveNet/RTMPose，演示为确定性回退）+ 国标折算分 + 弱项锻炼处方 | 只存关键点统计，原始视频推理后即丢弃 |
| 选修课推荐 | 六因子可解释打分：体测弱项+40(最高权) / 健康趋势±15 / 成绩扬长补短+20 / 兴趣+25 / 校外同项目排除 / 时段冲突硬约束 | 每条推荐附理由；探索推荐防信息茧房；家长可一键关闭个性化（原序返回）；报名时理由快照留痕 |
| 请假台账 | 病假症状标签 → 同班3天内同症状≥3例自动进多模态融合（R-ILLNESS 规则），生成缺勤聚集风险 | 班主任首次能看到每生本月/学期请假统计；症状脱敏聚合，不公开个人 |
| 用餐日历 | 按月勾选工作日 → 后厨按日备餐台账 → 月末按实际天数出账 | 后付费银行卡代扣、无余额无饭卡；请假获批只提示重叠用餐日，不自动动钱 |
| 年检公示墙 | 7门类效期规则引擎（甲醛/直饮水/消防/照明/体育器械/急救箱/课桌椅），超期/临期30天/不合格自动提醒 | 报告编号可核验；主动公开回应家长关切 |
| 课桌椅匹配 | GB/T 3976 型号身高区间 × 体检档案身高联查 | 只公示匹配比例与"X同学建议2号桌"，不公示身高数值 |
| 资产淘汰清单 | 购置年限超门类年限（课桌椅8年/体育器械6年）或成色≤2 自动入列 | 台账归口校长（安全第一责任人），教师端 403；扫码报修不受限 |
| 场馆口碑 | 教师带班点评，评分≤2星自动置顶"差评规避提示" | 仅教师/校长可写，家长 403 |
| 家庭安全提示 | 火/刀具/窗台等家庭场景提醒 | 只推家长通知，校园 AI 不进家庭 |

**慎做清单**（主动规避，作为答辩点）：不做人脸识别、不做情绪识别、不做成绩/体测排名公示、校园感知不延伸到家庭与校外水域。

### 核心技术点
- **本体论框架**（`app/ai/ontology.py`）：subClassOf 传递闭包、区域属性 A-Box、前向链融合规则；
  每个风险事件带完整推理链（信号→本体路径→规则→处置建议），可解释、可审计
- **多模态感知**：sensor / vision / citizen / official 四类模态，≥2 类独立模态交叉印证才确认风险（防误报）；
  两个例外有明确论证：地震官方预警高可信单源直通；缺勤聚集 R-ILLNESS 以"3份独立家长病假单"等价多源
- **新增双模态规则 R-STAIR**：楼梯间视觉密度 + 噪音≥75dB 交叉印证 → 拥挤风险（分层错时下楼预案）
- **边缘计算 + 数据安全**：视频帧在边缘侧推理后即丢弃，云端只存事件类型/bbox/SHA-256 快照哈希——无人脸、无原图；
  对齐《未成年人网络保护条例》
- **IoT 传感器**：水位/噪音(含楼梯间)/微气象/留样柜温度/甲醛TVOC/双停车场地磁，MQTT topic `redgate/{device_id}/{metric}`
- **预案引擎**：`(预警类型×级别) → 运行模式 + 动作清单`，模式单调升级防抖动，每次切换留审计痕，矩阵全量公开
- **图算法**：校园路网 Dijkstra，活跃风险区域动态加权（severity×500m 等效代价），实时改道
- **通知中枢**（对标并超越微信群）：定向到人、回执可查、紧急通知 15 分钟未读自动列入电话补呼清单

## 快速开始

```bash
cd backend
pip install fastapi uvicorn sqlalchemy pydantic httpx pytest
python seed.py                                   # 初始化全部演示数据
python -m uvicorn app.main:app --port 8000       # 前端由根路径直接服务
# 打开 http://127.0.0.1:8000
```

可选增强（不装也能完整演示）：
- `REDGATE_YOLO_MODEL=/path/yolov8n.onnx` + `pip install onnxruntime pillow numpy` → 真实视频推理
- `REDGATE_LLM_API_KEY=...` → 工单 LLM 分类
- `REDGATE_DB_URL=postgresql://...` → 切换 PostgreSQL

测试：`python -m pytest tests/ -q`（46 项：四层链路 + 平时线全功能 + 权限矩阵 + 场景端到端）

| 张主任 | 社区居委会·红门管家（资源方：只审批预约，不发起） | 工单处置台 / 风险事件 / 预约审批 |

权限设计要点：
- **一人一档、不一人一号**：小学生无手机、班级一体机无采购预算，故学生不设账号；
  学生档案（Student 挂 parent_id）由家长账号内嵌查看，三级守卫：本家长 / 同班班主任 / 校长
- **资产台账归口校长**：教师访问 `/api/assets` 返回 403；但**扫码报修任何角色可用**（发现坏损即报）
- **预约单向流**：学校侧（教师/校长）发起 → 社区侧审批（403 强制，接口层校验）
- **隐私边界真实**：种子数据中同班同学各挂独立家长账号，跨家长查档案 403 有真实语义

## 演示动线（评委 3 分钟）

1. **赵校长登录** → 指挥大屏（深色城运主题）：点场景注入器「**放学高峰暴雨**」：
   橙警接入 → 预案切"室内避险"（顶栏红闪）→ 水位计+AI积水+家长上报交叉印证 → 点开**推理链**
2. 再注入「**缺勤聚集**」：3份独立病假单（发热）→ R-ILLNESS 触发 → 校医排查预案；
   「**楼梯间拥挤**」：视觉密度+噪音 78dB 双模态 → 分层错时下楼
3. **王女士登录** → 孩子空间：小明 BMI 超重 → 营养建议（非医疗诊断声明）；视力三学期连降 → 提醒；
   跳绳 60 分弱项 → **选修课页"花样跳绳"置顶且附推荐理由**，校园篮球因校外已学被排除；上传视频体验 AI 体测
4. 家长提交病假（勾症状）→ 切**李老师**审批 → 弹出用餐重叠提示（只建议不动钱）+ 缺勤统计每生学期天数
5. 赵校长看**资产管理**（教师端无此入口）：六年级课桌成色最差、超龄淘汰清单自动生成；
   **年检公示墙**：灭火器超期红色提醒、课桌椅匹配率只公示比例不公示身高
6. 点「解除应急」→ 全系统回到正常运行
